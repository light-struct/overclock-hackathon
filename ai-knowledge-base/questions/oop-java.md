## 🟢 Лёгкие (`easy`)

---

### Q1: Какое ключевое слово используется для наследования в Java?

- **ID:** `oop-java-easy-01`
- **Difficulty:** easy
- **Tags:** наследование, синтаксис

**Варианты:**
- A) implements
- B) inherits
- C) super
- D) extends

**Ответ:** `D`

**Объяснение:** В Java для наследования используется ключевое слово `extends`: `class Dog extends Animal`. Ключевое слово `implements` используется для реализации интерфейсов, а `super` — для обращения к суперклассу.

---

### Q2: Сколько классов может наследовать Java-класс одновременно?

- **ID:** `oop-java-easy-02`
- **Difficulty:** easy
- **Tags:** наследование, ограничения-java

**Варианты:**
- A) Неограниченное количество
- B) Только один класс
- C) До трёх классов
- D) Только два класса

**Ответ:** `B`

**Объяснение:** Java поддерживает только одиночное наследование классов — класс может наследовать только один класс. Это сделано для избежания "diamond problem". Однако класс может реализовывать несколько интерфейсов.

---

### Q3: Что делает ключевое слово `final` применительно к классу?

- **ID:** `oop-java-easy-03`
- **Difficulty:** easy
- **Tags:** final, наследование

**Варианты:**
- A) Делает все поля класса константами
- B) Делает класс абстрактным
- C) Запрещает создание экземпляров класса
- D) Запрещает наследование от этого класса

**Ответ:** `D`

**Объяснение:** `final class` нельзя расширить (унаследовать). Например, `String` в Java — final класс. `final` на поле делает его константой. `final` на методе запрещает его переопределение в подклассе.

### Q4: Что такое абстрактный класс в Java?

- **ID:** `oop-java-easy-04`
- **Difficulty:** easy
- **Tags:** abstract, абстракция

**Варианты:**
- A) Класс без полей
- B) Класс только со статическими методами
- C) Класс, реализующий один интерфейс
- D) Класс, у которого нельзя создать экземпляр; может содержать абстрактные и обычные методы

**Ответ:** `D`

**Объяснение:** Абстрактный класс объявляется с `abstract`. Нельзя написать `new AbstractClass()`. Может содержать как абстрактные методы (без реализации), так и конкретные. Подкласс обязан реализовать все абстрактные методы.

### Q5: Что такое аннотация @Override?

- **ID:** `oop-java-easy-05`
- **Difficulty:** easy
- **Tags:** override, аннотации

**Варианты:**
- A) Указывает компилятору создать перегруженную версию метода
- B) Делает метод финальным
- C) Сообщает компилятору что метод переопределяет метод суперкласса; компилятор проверяет корректность
- D) Запрещает вызов метода из суперкласса

**Ответ:** `C`

**Объяснение:** `@Override` — это маркер для компилятора. Если метод не переопределяет ни один метод суперкласса (например, опечатка в имени), компилятор выдаст ошибку. Это предотвращает незаметные баги с неправильными именами методов.

---

### Q6: Какой модификатор доступа делает член класса доступным только внутри самого класса?

- **ID:** `oop-java-easy-06`
- **Difficulty:** easy
- **Tags:** модификаторы-доступа, инкапсуляция

**Варианты:**
- A) protected
- B) default (package-private)
- C) public
- D) private

**Ответ:** `D`

**Объяснение:** `private` — самый строгий модификатор. Доступен только внутри самого класса. `protected` — класс + подклассы + пакет. `default` (без модификатора) — только внутри пакета. `public` — везде.

### Q7: Что происходит при вызове super() в конструкторе?

- **ID:** `oop-java-easy-07`
- **Difficulty:** easy
- **Tags:** super, конструкторы, наследование

**Варианты:**
- A) Создаётся новый объект суперкласса в heap
- B) Вызывается конструктор родительского класса
- C) Вызывается статический метод суперкласса
- D) Возвращается ссылка на объект суперкласса

**Ответ:** `B`

**Объяснение:** `super()` вызывает конструктор родительского класса. Должен быть первой строкой в конструкторе подкласса. Если не написать явно, Java автоматически добавляет `super()` (вызов конструктора без аргументов суперкласса).

### Q8: Чем абстрактный класс отличается от интерфейса в Java (до Java 8)?

- **ID:** `oop-java-medium-01`
- **Difficulty:** medium
- **Tags:** abstract, interface, сравнение

**Варианты:**
- A) Интерфейс не может содержать никаких методов
- B) Абстрактный класс может иметь состояние (поля), конструкторы и конкретные методы; интерфейс — только абстрактные методы и константы
- C) Абстрактный класс не поддерживает полиморфизм
- D) Нет никакого отличия, это синонимы

**Ответ:** `B`

**Объяснение:** До Java 8 интерфейс содержал только абстрактные методы и `public static final` константы. Абстрактный класс может иметь поля, конструкторы, конкретные методы. С Java 8 интерфейсы получили default и static методы.

### Q9: Может ли интерфейс содержать default-методы (Java 8+)?

- **ID:** `oop-java-medium-02`
- **Difficulty:** medium
- **Tags:** interface, default-методы, java8

**Варианты:**
- A) Нет, интерфейс никогда не содержит реализации
- B) Да, default-методы имеют реализацию и наследуются реализующими классами
- C) Только если интерфейс помечен @FunctionalInterface
- D) Только private default-методы без тела

**Ответ:** `B`

**Объяснение:** С Java 8 интерфейсы могут содержать default-методы с реализацией. Это позволило добавлять новые методы в существующие интерфейсы без нарушения обратной совместимости. Класс может переопределить default-метод.

### Q10: Что такое casting (приведение типов) в Java?

- **ID:** `oop-java-medium-03`
- **Difficulty:** medium
- **Tags:** casting, типы, полиморфизм

**Варианты:**
- A) Создание копии объекта нового типа
- B) Явное или неявное преобразование ссылки одного типа к другому
- C) Вызов метода через интерфейс
- D) Инициализация переменной значением другого типа

**Ответ:** `B`

**Объяснение:** Upcasting (к суперклассу) — неявное, всегда безопасно: `Animal a = new Dog()`. Downcasting (к подклассу) — явное, может выбросить ClassCastException: `Dog d = (Dog) a`. Перед downcast лучше проверить `instanceof`.

### Q11: Что делает оператор instanceof?

- **ID:** `oop-java-medium-04`
- **Difficulty:** medium
- **Tags:** instanceof, casting, полиморфизм

**Варианты:**
- A) Создаёт экземпляр указанного класса
- B) Проверяет, является ли объект экземпляром указанного класса или его подкласса
- C) Сравнивает два объекта по значению полей
- D) Возвращает тип объекта в виде строки

**Ответ:** `B`

**Объяснение:** `obj instanceof ClassName` возвращает `true` если obj является экземпляром ClassName или его подкласса. Возвращает `false` если obj == null. С Java 16: `if (obj instanceof Dog d) { d.bark(); }` — pattern matching.

### Q12: Можно ли переопределить (override) статический метод в Java?

- **ID:** `oop-java-medium-05`
- **Difficulty:** medium
- **Tags:** static, override, полиморфизм

**Варианты:**
- A) Да, как и любой другой метод с @Override
- B) Нет, статические методы скрываются (method hiding), но не переопределяются
- C) Только если суперкласс abstract
- D) Только с явной аннотацией @Override

**Ответ:** `B`

**Объяснение:** Статические методы не участвуют в динамическом полиморфизме. Если в подклассе объявить метод с той же сигнатурой — это hiding (скрытие), а не overriding. Вызов будет определён по типу переменной во время компиляции, а не по типу объекта в runtime.

### Q13: Что такое вложенный (nested) класс в Java?

- **ID:** `oop-java-medium-06`
- **Difficulty:** medium
- **Tags:** nested-class, inner-class

**Варианты:**
- A) Класс, наследующий несколько интерфейсов
- B) Класс без публичного конструктора
- C) Класс, объявленный внутри другого класса
- D) Класс, реализующий интерфейс Cloneable

**Ответ:** `C`

**Объяснение:** Nested class — класс внутри класса. Виды: static nested class (нет доступа к полям внешнего), inner class (есть доступ к нестатическим полям внешнего), local class (внутри метода), anonymous class (без имени).

### Q14: Что такое covariant return type в Java?

- **ID:** `oop-java-medium-07`
- **Difficulty:** medium
- **Tags:** override, covariant-return, наследование

**Варианты:**
- A) Метод возвращает строго тот же тип, что и переопределяемый метод
- B) При переопределении метод в подклассе может возвращать подтип типа суперкласса
- C) Метод всегда возвращает Object
- D) Тип возвращаемого значения — параметр дженерика

**Ответ:** `B`

**Объяснение:** Covariant return type (с Java 5): при переопределении можно сузить тип возвращаемого значения. Если суперкласс возвращает `Animal`, подкласс может вернуть `Dog` (подтип Animal). Это не нарушает контракт суперкласса.

### Q15: Что происходит при downcast к несовместимому типу без проверки instanceof?

- **ID:** `oop-java-hard-01`
- **Difficulty:** hard
- **Tags:** casting, ClassCastException, исключения

**Варианты:**
- A) NullPointerException во время выполнения
- B) ClassCastException во время выполнения
- C) StackOverflowError
- D) Компилятор запрещает такую операцию во время компиляции

**Ответ:** `B`

**Объяснение:** Компилятор не всегда может проверить корректность downcasting — это проверяется в runtime. При несовместимых типах выбрасывается `ClassCastException`. Поэтому перед downcast проверяй: `if (animal instanceof Dog dog) { dog.bark(); }`.

### Q16: Что произойдёт, если не вызвать super() явно в конструкторе подкласса?

- **ID:** `oop-java-hard-02`
- **Difficulty:** hard
- **Tags:** super, конструкторы, наследование

**Варианты:**
- A) Компилятор выдаст ошибку компиляции
- B) Java автоматически вставляет вызов super() без аргументов первой строкой
- C) Объект создастся, но поля суперкласса останутся null/0
- D) Вызывается конструктор с наибольшим числом параметров суперкласса

**Ответ:** `B`

**Объяснение:** Если не написать `super()` явно, компилятор автоматически вставляет вызов `super()` (без аргументов) первой строкой. Если в суперклассе нет конструктора без аргументов — будет ошибка компиляции.

### Q17: В чём разница между anonymous class и lambda в Java 8+?

- **ID:** `oop-java-hard-03`
- **Difficulty:** hard
- **Tags:** anonymous-class, lambda, functional-interface

**Варианты:**
- A) Они полностью идентичны по поведению
- B) Lambda создаёт новый класс, anonymous — нет
- C) Anonymous class может реализовывать интерфейсы с несколькими методами и иметь состояние; lambda — только функциональные интерфейсы, без своего состояния и this
- D) Anonymous class работает быстрее lambda

**Ответ:** `C`

**Объяснение:** Anonymous class: может реализовывать любой интерфейс/абстрактный класс, иметь поля, `this` ссылается на сам anonymous объект. Lambda: только для функциональных интерфейсов (1 абстрактный метод), нет собственного состояния, `this` ссылается на окружающий класс.

### Q18: Что такое множественное наследование через интерфейсы и проблема Diamond в Java?

- **ID:** `oop-java-hard-04`
- **Difficulty:** hard
- **Tags:** interface, diamond-problem, default-методы

**Варианты:**
- A) Diamond problem в Java невозможна, так как интерфейсы не имеют реализации
- B) Если класс реализует два интерфейса с одинаковым default-методом — компилятор требует явного переопределения в классе
- C) Java автоматически выбирает default-метод из первого по алфавиту интерфейса
- D) Конфликт default-методов — ошибка runtime, а не компиляции

**Ответ:** `B`

**Объяснение:** С Java 8 (default-методы) diamond problem стала возможной. Если класс реализует два интерфейса A и B с одинаковым default-методом `foo()`, компилятор выдаёт ошибку и требует явно переопределить метод в классе: `@Override public void foo() { A.super.foo(); }`.

### Q19: Как работает method dispatch (диспетчеризация) в Java — static vs dynamic binding?

- **ID:** `oop-java-hard-05`
- **Difficulty:** hard
- **Tags:** полиморфизм, binding, JVM

**Варианты:**
- A) Все методы в Java используют static binding (compile-time)
- B) Static binding — для final/static/private методов (compile-time); dynamic binding — для overridden методов (runtime, по реальному типу объекта)
- C) Dynamic binding используется только для интерфейсных методов
- D) Static binding быстрее, поэтому JVM оптимизирует все методы в статические

**Ответ:** `B`

**Объяснение:** Static binding: тип определяется во время компиляции. Применяется к `static`, `final`, `private` методам и конструкторам. Dynamic binding: реальный метод определяется в runtime по типу объекта (через vtable в JVM). Это основа полиморфизма — `Animal a = new Dog(); a.speak()` вызовет `Dog.speak()`.

### Q20: Что такое маркерный интерфейс (marker interface) в Java и зачем он нужен?

- **ID:** `oop-java-hard-06`
- **Difficulty:** hard
- **Tags:** interface, marker-interface, Serializable

**Варианты:**
- A) Интерфейс с одним абстрактным методом (SAM)
- B) Интерфейс без методов, служащий тегом/меткой для JVM или библиотек
- C) Интерфейс, помеченный аннотацией @Marker
- D) Устаревший паттерн, полностью заменённый аннотациями

**Ответ:** `B`

**Объяснение:** Marker interface — пустой интерфейс без методов. Примеры: `Serializable`, `Cloneable`, `RandomAccess`. Служит "меткой" для JVM или библиотек: `instanceof Serializable` говорит, что объект можно сериализовать. Сейчас частично заменяются аннотациями, но `Serializable` по-прежнему активно используется.
