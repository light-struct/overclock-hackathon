### Q1: Что такое метод в Java?

- **ID:** `proc-easy-01`
- **Difficulty:** easy
- **Tags:** методы, основы

**Варианты:**
- A) Переменная, хранящая блок кода
- B) Именованный блок кода, выполняющий задачу и потенциально возвращающий значение
- C) Экземпляр класса с состоянием и поведением
- D) Специальный тип данных для хранения функций

**Ответ:** `B`

**Объяснение:** Метод — именованный блок кода, который выполняет определённую задачу. Может принимать параметры и возвращать значение (или void). Методы обеспечивают повторное использование кода и декомпозицию задачи на подзадачи.

### Q2: Что делает оператор break в Java?

- **ID:** `proc-easy-02`
- **Difficulty:** easy
- **Tags:** break, циклы, управление-потоком

**Варианты:**
- A) Приостанавливает выполнение программы на 1 секунду
- B) Пропускает текущую итерацию и переходит к следующей
- C) Завершает выполнение текущего цикла или switch и переходит к коду после них
- D) Выбрасывает исключение BreakException

**Ответ:** `C`

**Объяснение:** `break` немедленно завершает ближайший цикл (for/while/do-while) или switch и передаёт управление коду, следующему после него. Используется для досрочного выхода из цикла при выполнении условия.

### Q3: Что делает continue в цикле?

- **ID:** `proc-easy-03`
- **Difficulty:** easy
- **Tags:** continue, циклы, управление-потоком

**Варианты:**
- A) Завершает цикл полностью как break
- B) Пропускает оставшуюся часть текущей итерации и переходит к следующей
- C) Переходит к следующему оператору за пределами цикла
- D) Перезапускает цикл с самого начала (i=0)

**Ответ:** `B`

**Объяснение:** `continue` прерывает текущую итерацию цикла и переходит к следующей: в for-цикле выполняет шаг инкремента и проверку условия; в while/do-while — сразу к проверке условия. Полезно для пропуска ненужных элементов без выхода из цикла.

### Q4: Каков диапазон значений типа int в Java?

- **ID:** `proc-easy-04`
- **Difficulty:** easy
- **Tags:** примитивные-типы, int

**Варианты:**
- A) -2^63 до 2^63-1
- B) 0 до 4,294,967,295
- C) -32,768 до 32,767
- D) -2^31 до 2^31-1 (от -2,147,483,648 до 2,147,483,647)

**Ответ:** `D`

**Объяснение:** `int` — 32-битный знаковый целочисленный тип. Диапазон: -2^31 до 2^31-1 = от -2,147,483,648 до 2,147,483,647. `long` — 64-бит (-2^63 до 2^63-1). `short` — 16-бит (-32768 до 32767). `byte` — 8-бит (-128 до 127).

### Q5: Что такое массив в Java?

- **ID:** `proc-easy-05`
- **Difficulty:** easy
- **Tags:** массивы, основы

**Варианты:**
- A) Динамическая структура данных переменного размера

- D) Хэш-таблица пар ключ-значение

**Ответ:** `C`

**Объяснение:** Массив в Java — объект фиксированного размера, хранящий элементы одного типа. Индексация с 0. Размер задаётся при создании и не меняется. Доступ по индексу за O(1). При выходе за границы — `ArrayIndexOutOfBoundsException`.

### Q6: Какой оператор используется для получения остатка от деления?

- **ID:** `proc-easy-06`
- **Difficulty:** easy
- **Tags:** операторы, арифметика

**Варианты:**
- A) /
- B) //
- C) mod
- D) %

**Ответ:** `D`

**Объяснение:** `%` — оператор остатка от деления (modulo). `7 % 3 = 1`, `10 % 2 = 0`. Полезен для: проверки чётности (`n % 2 == 0`), циклической индексации (`i % size`), проверки делимости. В Java работает и с вещественными числами и с отрицательными числами.

### Q7: Чем отличается while от do-while?

- **ID:** `proc-easy-07`
- **Difficulty:** easy
- **Tags:** циклы, while, do-while

**Варианты:**
- A) while проверяет условие после тела, do-while — до
- B) do-while работает значительно быстрее при любых условиях
- C) while проверяет условие до выполнения тела; do-while — после (тело выполняется хотя бы один раз)
- D) Нет никакой разницы в поведении

**Ответ:** `C`

**Объяснение:** `while(condition){}` — сначала проверяет условие, тело может не выполниться ни разу. `do{ }while(condition)` — сначала выполняет тело, потом проверяет условие, тело выполняется минимум один раз. Do-while полезен когда нужно выполнить тело хотя бы раз (например, ввод данных).

### Q8: Как Java передаёт аргументы в методы — по значению или по ссылке?

- **ID:** `proc-medium-01`
- **Difficulty:** medium
- **Tags:** передача-параметров, pass-by-value

**Варианты:**
- A) Примитивы — по значению, объекты — по ссылке (как в C++)
- B) Всегда по значению: для примитивов копируется само значение, для объектов — копируется значение ссылки
- C) Всегда по ссылке для всех типов
- D) Зависит от модификатора доступа метода

**Ответ:** `B`

**Объяснение:** Java — всегда pass-by-value. Для примитивов: метод получает копию значения, изменение не влияет на оригинал. Для объектов: метод получает копию ссылки — может изменить состояние объекта через эту ссылку, но не может переназначить оригинальную переменную. `void setNull(Object o){ o = null; }` — не обнулит оригинал.

### Q9: Что такое перегрузка методов (method overloading)?

- **ID:** `proc-medium-02`
- **Difficulty:** medium
- **Tags:** overloading, методы, компиляция

**Варианты:**
- A) Определение метода с тем же именем в подклассе
- B) Несколько методов с одинаковым именем, но разными параметрами (тип, количество) в одном классе
- C) Вызов метода суперкласса через ключевое слово super
- D) Метод с переменным числом аргументов через varargs

**Ответ:** `B`

**Объяснение:** Overloading — несколько методов одного имени в одном классе с разными сигнатурами: `print(int)`, `print(String)`, `print(int, String)`. Компилятор выбирает нужный метод во время компиляции по типу аргументов (static dispatch). Тип возвращаемого значения не является частью сигнатуры.

### Q10: Что такое varargs в Java?

- **ID:** `proc-medium-03`
- **Difficulty:** medium
- **Tags:** varargs, методы, массивы

**Варианты:**
- A) Список переменных экземпляра класса
- B) Механизм передачи переменного числа аргументов одного типа: `void method(int... nums)`
- C) Аргументы с дефолтными значениями (как в Python)
- D) Именованные параметры методов

**Ответ:** `B`

**Объяснение:** Varargs (`Type... name`) позволяет передавать 0 и более аргументов одного типа. Внутри метода — обычный массив. `method(1, 2, 3)` или `method()` или `method(new int[]{1,2})`. Varargs должен быть последним параметром. Пример: `System.out.printf(String format, Object... args)`.

### Q12: Как работает switch-выражение в Java?

- **ID:** `proc-medium-05`
- **Difficulty:** medium
- **Tags:** switch, управление-потоком, fallthrough

**Варианты:**
- A) Выполняет все ветки подряд независимо от совпадения
- B) Выполняет ветку, совпадающую с переданным значением; без break происходит fallthrough на следующие ветки
- C) Работает только со строками и enum-значениями
- D) Всегда требует ветку default для компиляции

**Ответ:** `B`

**Объяснение:** switch выполняет ветку case с совпадающим значением. Без `break` происходит fallthrough — выполнение продолжается в следующем case. `default` выполняется если ни один case не совпал. С Java 14: `switch` как выражение (`var x = switch(y){ case A -> 1; case B -> 2; }`).

### Q13: Что такое статический метод и как его вызвать?

- **ID:** `proc-medium-06`
- **Difficulty:** medium
- **Tags:** static, методы

**Варианты:**
- A) Метод, который нельзя переопределить в подклассе
- B) Метод, принадлежащий классу, а не экземпляру; вызывается через имя класса без создания объекта
- C) Метод с константными параметрами, которые нельзя изменить
- D) Метод, работающий только с final-полями класса

**Ответ:** `B`

**Объяснение:** Static метод принадлежит классу. Вызов: `ClassName.method()`. Не имеет доступа к нестатическим полям/методам (нет `this`). Применение: утилитные методы (`Math.sqrt()`), фабричные методы (`Integer.valueOf()`), методы `main()`.

### Q14: Что произойдёт при делении целого числа на ноль в Java?

- **ID:** `proc-medium-07`
- **Difficulty:** medium
- **Tags:** операторы, исключения, арифметика

**Варианты:**
- A) Вернётся значение null
- B) Вернётся специальное значение NaN
- C) Компилятор запрещает такую операцию на этапе компиляции
- D) Выбрасывается ArithmeticException: / by zero

**Ответ:** `D`

**Объяснение:** Деление целых чисел на 0 → `ArithmeticException: / by zero`. Деление вещественных (`double`/`float`) на 0.0 → `Double.POSITIVE_INFINITY`, `-0.0 / 0.0 → NaN` (не исключение). Это соответствует стандарту IEEE 754 для чисел с плавающей точкой.

### Q15: Что такое хвостовая рекурсия и поддерживает ли её JVM?

- **ID:** `proc-hard-01`
- **Difficulty:** hard
- **Tags:** рекурсия, хвостовая-рекурсия, JVM

**Варианты:**
- A) Хвостовая рекурсия — рекурсия с несколькими базовыми случаями
- B) Рекурсивный вызов является последней операцией в методе; JVM не оптимизирует её автоматически (нет TCO)
- C) JVM полностью оптимизирует все рекурсивные вызовы с Java 8
- D) Хвостовая рекурсия невозможна в Java из-за строгой типизации

**Ответ:** `B`

**Объяснение:** Хвостовая рекурсия (tail recursion): рекурсивный вызов — последняя операция, результат передаётся напрямую. JVM не выполняет Tail Call Optimization (TCO) автоматически (в отличие от Scala/Kotlin с @tailrec). Поэтому глубокая рекурсия в Java всё равно вызовет StackOverflowError. Решение: итеративный вариант или trampolining.

### Q16: Как работает labeled break в Java?

- **ID:** `proc-hard-02`
- **Difficulty:** hard
- **Tags:** break, метки, вложенные-циклы

**Варианты:**
- A) Помечает цикл для оптимизации JIT-компилятором
- B) Позволяет выйти из конкретного внешнего цикла по метке, а не только из ближайшего
- C) Является устаревшим синтаксисом и удалён в Java 17
- D) Аналог goto из C/C++, позволяет прыгать в любое место кода

**Ответ:** `B`

**Объяснение:** `outer: for(...) { inner: for(...) { if(cond) break outer; } }` — выходит из цикла, помеченного `outer:`. Без метки `break` вышел бы только из внутреннего цикла. Полезно при работе с вложенными циклами для досрочного выхода из всей конструкции.

### Q17: В чём разница между передачей int[] и Integer[] в метод?

- **ID:** `proc-hard-03`
- **Difficulty:** hard
- **Tags:** массивы, autoboxing, примитивы

**Варианты:**
- A) Нет разницы — оба передаются по ссылке
- B) int[] — массив примитивов (нет null-элементов, быстрее по памяти); Integer[] — массив объектов-обёрток (может содержать null, autoboxing overhead)
- C) Integer[] нельзя передать в метод, принимающий Object
- D) int[] автоматически конвертируется в Integer[] при передаче

**Ответ:** `B`

**Объяснение:** `int[]` — массив примитивов, нет накладных расходов объекта. `Integer[]` — массив объектов Integer, каждый элемент — отдельный объект в heap. `int[]` нельзя передать как `Integer[]` и наоборот без явного преобразования. `Arrays.stream(int[])` → IntStream; `Arrays.stream(Integer[])` → Stream<Integer>.

### Q18: Как работает ternary оператор и каковы его ограничения?

- **ID:** `proc-hard-04`
- **Difficulty:** hard
- **Tags:** операторы, ternary

**Варианты:**
- A) Ternary — полный аналог if-else без каких-либо ограничений
- B) Условие ? значение_если_true : значение_если_false; оба результата должны быть совместимых типов; нельзя вызывать void-методы
- C) Ternary можно использовать только внутри методов типа boolean
- D) Ternary не поддерживает вложенность

**Ответ:** `B`

**Объяснение:** `condition ? expr1 : expr2` — возвращает expr1 если condition true, иначе expr2. Ограничения: оба выражения должны быть совместимых типов (или приводимых); нельзя вызывать void-методы как выражение. Вложенность возможна, но снижает читаемость. Тип результата определяется по правилам продвижения типов.

### Q19: Что такое integer overflow и как его избежать?

- **ID:** `proc-hard-05`
- **Difficulty:** hard
- **Tags:** примитивные-типы, overflow, Math

**Варианты:**
- A) Java автоматически конвертирует int в long при переполнении
- B) При превышении максимального значения int происходит молчаливое переполнение (wrap-around) без исключения; решение: использовать long или Math.addExact()
- C) JVM бросает ArithmeticException при переполнении int
- D) Integer.MAX_VALUE + 1 возвращает Double.POSITIVE_INFINITY

**Ответ:** `B`

**Объяснение:** `Integer.MAX_VALUE + 1 = Integer.MIN_VALUE` — тихое переполнение, никакого исключения. Решения: 1) использовать `long` (`2147483647L + 1`). 2) `Math.addExact(a, b)` — бросает ArithmeticException при переполнении. 3) `BigInteger` для произвольно больших чисел.

### Q20: Как работает инициализация static блока и каков порядок инициализации класса?

- **ID:** `proc-hard-06`
- **Difficulty:** hard
- **Tags:** static, инициализация, загрузка-класса

**Варианты:**
- A) Static блок выполняется при каждом создании нового объекта класса
- B) Static блок выполняется один раз при первой загрузке класса JVM; порядок: static поля и блоки в порядке объявления
- C) Static блок выполняется после конструктора первого объекта
- D) Порядок выполнения static блоков недетерминирован

**Ответ:** `B`

**Объяснение:** Порядок инициализации класса: 1) Загрузка класса загрузчиком. 2) Static поля инициализируются значениями по умолчанию. 3) Static поля и static{} блоки выполняются в порядке появления в коде. 4) При `new`: поля экземпляра (дефолты), затем инициализаторы и конструктор. Static блок выполняется ОДИН раз на всё время работы JVM.
