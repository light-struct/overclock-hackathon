## 📋 Формат вопроса

Каждый вопрос в базе имеет строго фиксированный формат.  
**Не нарушайте структуру** — AI парсит поля по заголовкам.

```markdown
### Q{номер}: {Текст вопроса}

- **ID:** {topic_id}-{difficulty}-{порядковый_номер}
- **Difficulty:** easy | medium | hard
- **Tags:** тег1, тег2

**Варианты:**
- A) {Вариант A}
- B) {Вариант B}
- C) {Вариант C}
- D) {Вариант D}

**Ответ:** `{A|B|C|D}`

**Объяснение:** {Развёрнутое объяснение правильного ответа. Минимум 1–3 предложения.}

---
```

### Поля вопроса

| Поле | Обязательное | Описание |
|------|:---:|---------|
| `### Q{N}: текст` | ✅ | Номер и текст вопроса |
| `ID` | ✅ | Уникальный идентификатор: `{topic}-{difficulty}-{N}` |
| `Difficulty` | ✅ | `easy` / `medium` / `hard` |
| `Tags` | ✅ | Ключевые слова для фильтрации (через запятую) |
| `Варианты A–D` | ✅ | Ровно 4 варианта, один правильный |
| `Ответ` | ✅ | Одна буква в backticks: \`A\`, \`B\`, \`C\` или \`D\` |
| `Объяснение` | ✅ | Объяснение для студента после теста |

## ⚙️ Правила редактирования

### Добавление нового вопроса
1. Открой нужный файл в `questions/`
2. Найди нужный раздел сложности (`## 🟢 Лёгкие`, `## 🟡 Средние`, `## 🔴 Сложные`)
3. Скопируй шаблон вопроса, заполни все поля
4. Убедись что `ID` уникален: `{topic}-{difficulty}-{N}` где N — следующий номер
5. Проверь что правильный ответ соответствует полю `Ответ`

### Правила нумерации ID

```
oop-easy-01       ← ООП, лёгкий, первый
oop-medium-07     ← ООП, средний, седьмой
java-hard-03      ← Java Core, сложный, третий
gc-easy-05        ← Garbage Collector, лёгкий, пятый
```

## 🔢 Настройки теста

AI генерирует тест по следующим параметрам пользователя:

```yaml
Тема:        [выбор из доступных тем]
Кол-во вопросов: [3 | 5 | 10 | 15]
Сложность:   [easy | medium | hard]
```

### Распределение вопросов при adaptive mode
```
60% — вопросы из слабых тегов студента (из его истории ошибок)
40% — случайные вопросы по выбранной теме и сложности
```
